I
绪　论
板球系统的控制难题 · 国内外研究进展 · 本文技术路线

研究背景与意义

控制理论视角

板球系统的三层控制难题

  • 非线性：小球在斜面上的加速度由重力分量决定，而重力分量的作用方向随小球当前位置变化，系统响应不具有比例叠加性。
  • 强耦合：X 方向倾角调节不仅影响 X 向运动，还通过台面几何约束牵动 Y 方向；反之亦然，双轴难以完全解耦为两个独立通道。
  • 欠驱动：系统可直接控制的物理量仅有台面两个倾角，但需间接调控的状态量至少为四个（X、Y 方向的位置与速度），控制输入少于待控自由度。

国内外研究现状

国外研究进展

  • Hauser、Sastry 与 Kokotovic（1992）首次推导板球系统近似输入输出线性化形式，奠定了非线性控制研究的基础框架。
  • TecQuipment 将板球系统商品化为 CE151 实验平台，使其从论文方程变为可操作的物理对象。
  • Mochizuki 与 Ichihara 从广义 KYP 引理出发，将 I-PD 控制器参数设计转化为可系统求解的优化问题。
  • 近年趋势：从"能否稳定"转向"噪声、滞后、扰动共存时能否既稳又准"。

国内研究进展

  • 滕树杰和张乃尧给出分层模糊控制器的形式化解析表达；苏金涛等验证监督模糊策略在轨迹控制中的有效性。
  • 高多（东北大学）从伺服补偿器与动态输出反馈框架设计控制器；武凡凯（长春工业大学）在同一平台上完成三种 PID 策略的定量对比。
  • 尹梦碟（广西科大）以 OPENMV 搭建低成本平台并验证串级 PID——与本文技术路线高度相似，为本文重要参考。
  • 总体趋势：从纯仿真阶段过渡到嵌入式视觉、低成本硬件、实物闭环验证三者并行。

现有工作的缺口与本文切入点

部分工程文献对视觉测量误差的物理来源（像素量化、阈值漂移、运动模糊、通信时延）讨论不充分；仿真参数到实物参数的映射分析不足。本文不做新算法，聚焦于系统集成验证：以嵌入式视觉替代传统架构，在同一套参数下完成仿真到实物的迁移，并定量分析两者差距的来源。

动力学建模

系统组成与建模条件

板球系统的物理链路：步进电机 → 摇臂连杆 → 万向节 → 平台倾斜 → 小球滚动。在平台倾角 ≤ 5°、电机未失步的条件下，以下近似成立：

  • 小球纯滚动（不打滑、不跳离）；X/Y 两轴小角度解耦
  • sin θ ≈ θ，cos θ ≈ 1；摩擦与延迟等效为统一阻尼项

控制设计的关键约束：分母含 (z−1)²——两个极点位于单位圆上，开环临界稳定、无阻尼。电机串联后共三个状态。单环位置 PID 同时面对"快"与"稳"的冲突——串级 PID 的必要性根源于此。

设计原则与整定策略

  • 由内向外逐层闭合：先整定角度内环（带宽最高），确保执行机构对角度指令的快速跟随；再整定速度中环（施加阻尼抑制超调）；最后整定位置外环（保障稳态精度）。
  • 带宽递减 3~5 倍：内环 ~100 rad/s → 中环 ~10–20 rad/s → 外环 ~ 几 rad/s。各层带宽错开可有效减少环间耦合。
  • 速度信号获取：相邻采样周期的位置差分近似，串联一阶低通滤波器 0.2/(1−0.8z⁻¹) 以抑制差分运算引起的高频噪声放大。
  • 双轴初始共用同一套 PID 参数，以简化对比并为后续分轴独立整定提供基准。

软件系统

双端分工架构

视觉端 (MicroPython)：图像采集 → LAB 分割 → 连通域筛选 → 质心计算 → 串口发送。循环周期 19 ms (~53 Hz)。

主控端 (C，20 ms 定时中断)：

  1. 读 MaixCam 串口 → 校验 0x55 → 提取小球坐标
  2. 读 JY931 串口 → 解析平台实时倾角
  3. 位置误差 → 外环(目标速度) → 中环(目标角度) → 内环(脉冲频率)
  4. 限幅 → 更新 DIR 方向 → 定时器输出 STEP 脉冲

时序与保护机制

  • 视觉 19 ms / 控制 20 ms，采用"取最新帧"策略，不等待同步。
  • 输入限幅：相邻帧坐标差 > 30 px 视为无效，沿用上一帧值。
  • 输出限幅：PWM 频率卡上下界，防止共振或失步。
  • 通信超时：连续丢帧触发减速保护，不按过期数据持续驱动。

联调验收标准

以下四项全部满足后，方可进入性能测试环节，以预先排除硬件与通信层面的问题：

  • 通信稳定：无连续丢帧或错误帧，帧头 0x55 校验通过。
  • 周期稳定：主控定时中断 20 ms 不被阻塞，控制周期无拉长。

仿真与实物差异分析

差异一 · 传感器

像素量化引入测量噪声

仿真中坐标连续、精度理想；实物中 MaixCam 分辨率为 256×256，存在最大 ±0.5 像素的量化偏差。该噪声混入外环位置误差，经速度差分放大后传入内环，引起执行端高频微动。

差异二 · 执行器

转矩脉动替代理想平滑力矩

仿真假设电机输出力矩连续平滑；实物步进电机存在齿槽效应与非正弦驱动电流波形。低速末端控制量微小时，脉动分量使小球在目标点附近微颤，定点段速度呈偶发脉冲而非连续衰减。

差异三 · 传动机构

机械不对称破坏双轴同构假设

仿真中两轴摩擦与间隙参数对称；实物中连杆长度、万向节装配松紧、摩擦分布均存在加工公差内的差异。同构 PID 参数下 Y 轴速度摆幅（302.4 mm/s）较 X 轴（255.4 mm/s）高出约 18%。

差异四 · 通信链路

丢帧与周期异步引入局部滞后

仿真中数据准时到达、无丢失；实物中视觉端 19 ms / 控制端 20 ms 未做同步，实测丢帧率 0.07%（3004 帧中丢 2 帧）。需以超时降级策略兜底，避免过期数据持续驱动执行机构。

核心认识

控制精度的最终上限由视觉测量、执行机构、机械结构和控制器四方共同决定。低成本平台上，执行端与机械端的物理约束往往是真正的瓶颈所在。

结论与展望

1. 低成本硬件闭环方案可行

MaixCam + MSPM0G3507 + JY931 + D36A + 42 步进电机可支撑完整的视觉反馈闭环。检测有效率 99.93%，控制周期全程稳定 50 Hz，成本不足传统工业相机+PC 方案的十分之一。

2. LAB 色彩空间检测有效

亮度与色度分离使白球阈值分割对光照波动的容忍度优于 RGB 和 HSV。三特征评分筛选可有效排除反光、阴影等干扰，单帧处理时间小于 19 ms。

3. 串级 PID 可迁移至实物平台

同一套参数从仿真迁移至实物后，基本的收敛和跟踪能力得以保留。仿真与实物之间的差距具有明确的物理来源（摩擦、间隙、转矩脉动），而非算法失效。

4. 精度上限受执行端与机械端制约

定点段速度尖峰源于静摩擦切换；圆轨迹 Y 轴额外 18% 速度摆幅源于机械不对称与同构参数。算法、执行机构、机械结构三方共同决定系统性能边界。

后续改进方向
