国内外研究现状与本文切入点

国外研究：理论建模与标准平台较成熟

  • Hauser 等基于近似输入输出线性化处理板球系统。
  • TecQuipment 等推出商品化实验平台，便于教学与算法验证。
  • 部分研究利用广义 KYP 引理等方法系统求解 I-PD 参数。

国内研究：控制策略与低成本实现并行推进

  • 已有分层模糊控制、伺服补偿器、PID 策略对比等研究。
  • OpenMV 等视觉模块被用于低成本平台搭建。
  • 实物平台常受机械摩擦、传动间隙、视觉噪声影响。

本文切入点

在不追求复杂算法堆叠的前提下，重点完成一套可落地的低成本硬件系统，并解释仿真和实物之间的差异：算法能否收敛是一方面，视觉分辨率、执行器脉动和机械对称性同样决定最终精度。

研究背景与方案定位

为什么选择板球系统

典型非线性、强耦合、欠驱动对象

  • 非线性：小球加速度由台面倾角和当前位置共同决定。
  • 强耦合：X/Y 两轴通过台面几何约束和球面运动相互影响。
  • 欠驱动：仅能控制两个平台倾角，却需要调节位置与速度等状态量。
板球系统复杂度高于普通二阶对象，又比四旋翼等系统更易搭建，适合开展"建模—仿真—实物验证"的本科毕设。

系统建模与控制器设计

建模假设与对象特性

  • 小球纯滚动，台面小角度运动，近似满足 sinθ≈θ。
  • 摩擦、延迟与噪声统一等效为阻尼和扰动项。
  • 小球位置对平台倾角呈双积分特性，开环临界稳定。
  • 步进电机与传动机构近似为一阶惯性环节。
X 轴传动链结构示意图

为什么采用三环串级 PID

  • 单位置环难以同时兼顾快速性、稳定性和超调量。
  • 外环控制位置，中环约束速度，内环快速校正平台倾角。
  • 结构清晰、参数可调、便于从仿真迁移到实物平台。

工程实现中的保护策略

  • 位置与速度输入低通处理，抑制视觉噪声和差分放大。
  • 目标倾角和脉冲频率限幅，避免执行机构过冲。
  • 通信超时进入降级策略，防止丢帧导致平台失控。
  • 按固定周期取最新视觉帧，减少积压数据引入的滞后。

软件系统

双端分工架构

视觉端 (MicroPython)：图像采集 → LAB 分割 → 连通域筛选 → 质心计算 → 串口发送。循环周期 19 ms (~53 Hz)。

主控端 (C，20ms 定时中断)：

  1. 读 MaixCam 串口 → 校验 0x55 → 提取小球坐标
  2. 读 JY931 串口 → 解析平台实时倾角
  3. 位置误差 → 外环(目标速度) → 中环(目标角度) → 内环(脉冲频率)
  4. 限幅 → 更新 DIR 方向 → 定时器输出 STEP 脉冲

时序策略与保护机制

  • 视觉 19ms / 控制 20ms —— 每周期取缓冲区最新帧，不等待
  • 输入限幅：相邻帧坐标差 > 30px 视为无效，沿用上一帧值
  • 输出限幅：PWM 频率卡上下界，防止共振或失步
  • 通信超时：连续丢帧触发减速保护，不按过期数据持续驱动

仿真与实物差异分析

差异一 · 传感器

像素量化引入测量噪声

仿真中坐标连续、精度理想；实物中 MaixCam 分辨率为 256×256，存在最大 0.5 像素的量化偏差。噪声混入外环位置误差，经速度差分放大后传入内环，引起执行端高频微动。

差异二 · 执行器

转矩脉动替代理想平滑力矩

仿真假设电机输出力矩连续平滑；实物步进电机存在齿槽效应与非正弦驱动电流波形。低速末端控制量微小时，脉动分量使小球在目标点附近微颤，定点段速度呈偶发脉冲。

差异三 · 传动机构

机械不对称破坏双轴同构假设

仿真中两轴摩擦和间隙参数对称；实物中连杆长度、万向节装配松紧、摩擦分布均不同。同构 PID 参数下 Y 轴速度摆幅（302.4 mm/s）较 X 轴（255.4 mm/s）高出约 18%。

差异四 · 通信链路

丢帧与周期异步引入局部滞后

仿真中数据准时到达、无丢失；实物中视觉端 19ms / 控制端 20ms 未做同步，实测丢帧率 0.07%（3004 帧中丢 2 帧）。需以超时降级策略兜底，避免过期数据持续驱动。

核心认识

控制精度的最终上限由视觉测量、执行机构、机械结构和控制器四方共同决定。低成本平台上，执行端与机械端约束往往是主要瓶颈。

结论与展望

1. 完成低成本硬件闭环

搭建 MaixCam + MSPM0 + JY931 + 步进电机的平台，实现视觉反馈、控制计算和执行输出的完整闭环。

2. 验证 LAB 白球检测方案

LAB 色彩空间将亮度与色度解耦，相比 RGB/HSV 更适合白球阈值分割，单帧处理时间小于 19 ms。

3. 串级 PID 具备可迁移性

仿真与实物均表现出基本收敛和轨迹跟踪能力，说明所设计控制结构具备工程可用性。

4. 明确精度瓶颈来源

实物偏差主要来自像素量化、静摩擦、转矩脉动与机械不对称，而非单一控制算法失效。

后续改进方向

  • 视觉端：自适应 LAB 阈值与卡尔曼滤波帧间跟踪。
  • 控制端：分轴独立整定，引入摩擦观测与前馈补偿。
  • 算法端：补充滑模、反步法等策略横向对比。
  • 实验端：增加方形轨迹、Lissajous 曲线等基准测试。
